# Правила стратегического мышления

## Контекст → Анализ → Рекомендации

Каждое решение в системе проходит три стадии:

1. **Контекст** — Что известно на момент принятия решения?
2. **Аниз** — Какие есть варианты и их последствия?
3. **Рекомендация** — Какой путь выбран и почему?

## Компромиссы (Trade-offs)

| Решение | Преимущество | Риск |
|---------|--------------|------|
| Google Sheets как БД | Быстрый старт, видимость | Лимиты API, не для больших объёмов |
| Cron-оркестрация | Простота | Нет retry-логики из коробки |
| Полуавто-публикация | Контроль качества | Требует ручного approved=Y |
| Локальное хранение обложек | Контроль файлов | Нужен sync для CDN/доступа |

## Структурированные модели

### Агент (Unit Pattern)
Вход → Обработка → Выход + Лог


### Пайплайн (Pipeline Pattern)
Scheduler → [Агент 1-6] → Telegram


### Ошибка (Error Handling)
try → except → log → continue/fail


## Стандарты ответов

- Без наполнителей
- Без повторений
- Только структурированное рассуждение
