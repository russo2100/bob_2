# Границы системы

## Что входит в scope

| Компонент | В scope | Примечание |
|-----------|---------|------------|
| RSS парсинг | ✅ | 5 агентств, 24 часа |
| Perplexity Sonar API | ✅ | 5 брендов, 3-5 событий |
| Google Sheets интеграция | ✅ | NewsRaw + Texts |
| Генерация трендов | ✅ | TOP-5 в trends.md |
| Копирайтинг (Bob 2.0) | ✅ | 4 поста, 600-800 символов |
| Генерация обложек | ✅ | DALL-E/MJ/SD API |
| Telegram публикация | ✅ | sendPhoto + caption |
| Логирование | ✅ | Все этапы в Texts/progress |

## Что НЕ входит в scope

| Компонент | Вне scope | Причина |
|-----------|-----------|---------|
| Веб-интерфейс | ❌ | MVP без UI |
| База данных (PostgreSQL) | ❌ | Sheets достаточно для старта |
| A/B тесты постов | ❌ | Фаза 2 |
| Мультиязычность | ❌ | Только русский |
| Мобильное приложение | ❌ | Только Telegram |

## Границы данных

- **Не принимать за истину** данные без верификации (источник обязателен)
- **Не хранить** токены/секреты в репозитории
- **Не публиковать** без approved=Y (кроме тестового режима)

## Обработка неясностей

Если указание неясно:
1. Зафиксировать неопределённость в логе
2. Запросить уточнение у пользователя
3. Не предполагать значения по умолчанию для критичных параметров
